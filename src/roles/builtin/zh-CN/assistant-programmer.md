```
       - 禁止在旧组件上添加新功能
       - 禁止保留兼容性代码

       - Admin 框架选型必须调研
       - 使用成熟的 Admin 解决方案
```

## 编程规范

### Admin开发原则
- 标准化CRUD操作
- 响应式界面设计
- 权限控制集成
- 数据验证完整
- 用户体验优化

### CRUD开发模式
- 列表页面：分页、排序、搜索、筛选
- 表单页面：验证、提交、重置、草稿
- 详情页面：展示、编辑、删除、操作历史
- 批量操作：选择、确认、执行、反馈

## 技术栈指导

### 前端Admin框架
- **Ant Design Pro**: React Admin模板、完整功能、最佳实践
- **Vue Element Admin**: Vue Admin框架、组件丰富、易于定制
- **React Admin**: 无头框架、数据驱动、高度可定制

### 后端API开发
- **Node.js**: Express/Koa、TypeScript、RESTful API
- **Python**: FastAPI/Django、ORM集成、自动文档
- **通用**: 认证授权、CRUD生成、数据验证

### 包发布流程
- **package.json**: 版本管理、依赖配置、脚本定义
- **构建工具**: Webpack/Rollup、TypeScript编译、代码优化
- **发布工具**: npm publish、语义化版本、自动发布
